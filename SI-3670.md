M.scala
{code}
object M extends Application {
  def foo {
    lazy val a = {
        lazy val b = 1
        lazy val c = 2
        println(b)
        println(c)
    }
    a
    lazy val d = 42
    println(d)
  }
  
  foo
}
{code}

{code}
scalac M.scala
scala M

1
0
42
{code}

But hey, why is there 0 you ask? Well, all vals here share the same bitmap but b and c get also the same bit because of a bug in LazyVals line 51. Now if we fix that, with the example above we get
{code}
1
2
0
{code}
because now c and d share the same bit.
(In r23322) Closes SI-3670, SI-3877. Plus with some additional lazy val checking we avoid adding bitmaps that are overshadowed by other bitmaps. Review by dragos.
